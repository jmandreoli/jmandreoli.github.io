class PYTOOLS.experiment.Run(*args, **kwargs)[source]

Bases: Dispatcher

An instance of this class is a callable, taking no argument and returning no value. The execution of a run only changes its attribute values. All the keyword arguments of the constructor are passed to method setup().

Runs may emit messages during their execution. Callbacks attached to each message are passed a single argument, which is the run itself. Listeners (instances of class RunListener) define sets of consistent callbacks. A run can invoke a method add<name>Listener() to attach a listener of type <name> to itself. Some such methods are defined in class Run and its subclasses (see their documentation for details).

A typical event callback stores the result of a campaign of measures on the run. Measures (instances of class Measure) are in charge of reporting individual measures. A run can invoke a method add<name>Measure() to attach a measure of type <name> to itself. Some such methods are defined in class Run and its subclasses (see their documentation for details). In particular, all the pytorch losses can be added in this way.

Attributes (*) must be explicitly instantiated at creation time.

classmethod measure_factory(name, norm=None, doc=True)[source]

Used as decorator to attach a measure factory (the decorated object, assumed callable) to a subclass cls of this class. The factory is then available as method add<name>Measure() using the provided name. Its invocation accepts a parameter label under which its result in stored in the measures registry (by default, the label is the lowercase version of name).

loaders(pcval, **ka)[source]

Returns a dictionary which can be passed as keyword arguments to the Run constructor to initialise its required attributes:

  • non visible attributes train_data, valid_data, test_data, obtained by calling torch.utils.data.DataLoader on the corresponding split with key-word parameters ka (with prefix _ removed from keys);

  • a visible attribute data holding a description of the datasource;

  • all the attributes (visible or not) in ka, so they appear as attributes of the run.

Parameters:

pcval (float) – proportion (between 0. and 1.) of instances from the train split to use for validation.

display(rowspec, colspec, dataset=None, **ka)[source]

Displays a set of labelled data instances dataset in a grid. The specification of the rows (resp. cols) of the grid consists of three numbers:

  • height (resp. width) of the cells; required

  • number of rows (resp. cols); if -1 it is adjusted so the the whole dataset fits in the grid; default: 1

  • additional space in the row (resp. col) dimension; default: 0.

At most one of the number of rows or columns can be -1. When both are positive and the dataset does not fit in the grid, a slider is created to allow page browsing through the dataset.

Parameters:

  • rowspec,colspec – specification of the rows/cols of the grid

  • dataset (Iterable[Tuple[Any, int]]) – the data to display

  • ka – keyword arguments passed to matplotlib.pyplot.subplots() to create the grid
