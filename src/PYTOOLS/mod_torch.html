This module provides some utilities for pytorch development.

class PYTOOLS.torch.GeneralisedConvolution(K, P, Q=None, D=None, bias=True)[source]

Bases: Module

An instance of this class is a generalised convolution module. Ref:

Andreoli, Jean-Marc. 2019. ‘Convolution, Attention and Structure Embedding’. In Proc. of NeurIPS Workshop on Graph Representation Learning, 8. Vancouver, BC, Canada.

Model parameters are

\[\Theta^{\textrm{(x)}}{:}\langle K,P,D \rangle\; \Theta^{\textrm{(y)}}{:}\langle K,Q,D \rangle\; \Theta^{\textrm{(o)}}{:}\langle K,D \rangle\; \Theta^{\textrm{(oo)}}{:}\langle Q \rangle\]
Parameters:

  • K (int) – number of heads

  • P (int) – input dimension \(P\)

  • Q (int) – output dimension \(Q\), default \(P\)

  • D (int) – convolution head dimension \(D\), default \(\lfloor\frac{Q}{K}\rfloor\)

  • bias (bool) – whether to use the biases \(\Theta^{\textrm{(o),(oo)}}\)

temperature = None

Temperature of the softmax applied to scores

K: int

Number of heads

forward(score, x, mask=None, process_attn=<function GeneralisedConvolution.<lambda>>)[source]

Generalised Convolution formula (with biases):

\[\begin{split}\begin{align*} y_b & = \sum_k A_{bk}^\top\bar{x}_{bk}\Theta_k^{\textrm{(y)}\top}+\mathbf{1}_N\otimes\Theta^{\textrm{(oo)}}\\ \textrm{where } & A_{bk}{:}\langle M,N \rangle\; \bar{x}_{bk}{:}\langle M,D \rangle\\ A_{bk} & = \textrm{softmax}_{\textrm{col}}\frac{1}{\textrm{temp}}(\bar{A}_{bk}+\textrm{mask}_b)\\ \bar{x}_{bk} &= x_b \Theta_k^{\textrm{(x)}}+\mathbf{1}_M\otimes\Theta_k^{\textrm{(o)}} \end{align*}\end{split}\]
Parameters:

  • score – tensor \(\bar{A}{:}\langle B,K,M,N \rangle\) (attention scores)

  • x – tensor \(x{:}\langle B,M,P \rangle\) (input)

  • mask – tensor \(\langle B,M,N \rangle\) or \(\langle M,N \rangle\) (in log domain: possible values include \(-\infty\))

  • process_attn (Callable) – applied to the attention tensor \(\langle B,K,M,N \rangle\) (caution: large) to produce the auxiliary output; default returns None

Returns:

pair of output tensor \(y{:}\langle B,N,Q \rangle\) and auxiliary output (see process_attn)

Vaswani et al. 2017. ‘Attention Is All You Need’. arXiv: 1706.03762

Additional model parameters for Vanilla attention are:

\[\Lambda^{\textrm{(x)}}{:}\langle K,P',D' \rangle\; \Lambda^{\textrm{(y)}}{:}\langle K,Q',D' \rangle\; \Lambda^{\textrm{(o)}}{:}\langle K,D' \rangle\]
Parameters:

  • Qʹ (int | None) – query-input dimension \(Q'\), default \(Q\)

  • Dʹ (int | None) – attention head dimension \(D'\), default \(\lfloor\frac{Q'}{K}\rfloor\)

  • bias (bool) – whether to use the bias \(\Lambda^{\textrm{(o)}}\) + the convolution biases

  • args – passed to GeneralisedConvolution constructor

  • kargs – passed to GeneralisedConvolution constructor

forward(yʹ, xʹ=None, x=None, **kargs)[source]

Computes the attention scores passed to generalised convolution; include biases, not described in the paper, to conform with torch.nn.MultiheadAttention:

\[\begin{split}\begin{align*} \bar{A}_{bk} & = \bar{x}_{bk}\bar{y}_{bk}^\top\\ \textrm{where } & \bar{x}_{bk}{:}\langle M,D' \rangle,\; \bar{y}_{bk}{:}\langle N,D' \rangle\\ \bar{x}_{bk} & = x'_b\Lambda_k^{\textrm{(x)}}\\ \bar{y}_{bk} & = y'_b\Lambda_k^{\textrm{(y)}}+\mathbf{1}_N\otimes\Lambda_k^{\textrm{(o)}} \end{align*}\end{split}\]
Parameters:

  • yʹ – tensor \(y'{:}\langle B,N,Q' \rangle\) (query-input)

  • xʹ – tensor \(x'{:}\langle B,M,P' \rangle\) (key-input), default \(y'\)

  • x – passed to GeneralisedConvolution.forward() (value-input), default \(x'\)

  • kargs – passed to GeneralisedConvolution.forward()

Returns:

see GeneralisedConvolution.forward()

\[\begin{split}\begin{align*} \bar{A}_{bk} & = \bar{x}_{bk}\bar{y}_{bk}^\top+E_{\frac{md,mnd}{mn}}(\bar{x}_{bk},\bar{z}^{\textrm{(y)}}_{bk})+E_{\frac{nd,mnd}{mn}}(\bar{y}_{bk},\bar{z}^{\textrm{(x)}}_{bk})\\ \textrm{where } & \bar{x}_{bk}{:}\langle M,D' \rangle,\; \bar{y}_{bk}{:}\langle N,D' \rangle,\; \bar{z}_{bk}{:}\langle M,N,D' \rangle\\ \bar{x}_{bk} & = x'_b\Lambda_k^{\textrm{(x)}}+\mathbf{1}_M\otimes\Lambda_k^{\textrm{(ox)}}\\ \bar{y}_{bk} & = y'_b\Lambda_k^{\textrm{(y)}}+\mathbf{1}_N\otimes\Lambda_k^{\textrm{(oy)}}\\ \bar{z}^{\textrm{(x)}}_{bk} & = z'_b\Lambda_k^{\textrm{(zx)}}\\ \bar{z}^{\textrm{(y)}}_{bk} & = z'_b\Lambda_k^{\textrm{(zy)}}+\mathbf{1}_M\otimes\mathbf{1}_N\otimes\Lambda_k^{\textrm{(o)}} \end{align*}\end{split}\]

class PYTOOLS.torch.Einsum(sig, *dims, bias=True)[source]

Bases: Module

An instance of this class is essentially a Linear module allowing more flexibility in indices using the einsum notation. For example:

lin = torch.nn.Linear(3,4); lin_ = Einsum.torch_convert(lin)
assert lin_.sig == 'pq,bnp->bnq'
x = torch.rand(10,12,3) # batch:10, length:12
assert torch.all(lin(x)==lin_(x)) # if fails, try torch.allclose(lin(x),lin_(x))
Parameters:

  • sig (str) – an einsum-like signature conforming to sig_pattern

  • dims (int) – list of dimensions; matched to the weight component of sig

  • bias (bool) – whether to use a bias (on the intersection of the weight and output components of sig)

sig_pattern = re.compile('([a-z]+),([a-z]+)->([a-z]+)')

Pattern for allowed signatures (the 3 groups specify the weight, input and output components)

forward(x)[source]

Invokes torch.einsum() with signature given by attribute sig and arguments given by attribute weight and x, then adds the bias (attribute bias) if present.

static torch_convert(a)[source]

Converts a torch.nn.Linear instance into an instance of this class with same behaviour. The created instance has an additional method test_() which returns a comparison of the two modules on some sample given its batch size and sequence length.

Parameters:

a (Linear) – the instance to convert

Returns:

an equivalent Einsum instance
