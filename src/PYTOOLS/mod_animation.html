PYTOOLS.animation — Animation utilities

This module provides basic utilities to play animations in matplotlib. An animation is any callback which can be passed a simulation time, enumerated by a player object. The set of simulation times is split into a sequence of contiguous intervals called tracks. A track map is a function of simulation time which returns the bounds of its track interval if the simulation time is valid, and None otherwise.

Available types and functions

PYTOOLS.animation.track_function(track, stype=<class 'float'>)[source]

Builds a track map from a specification track. A track map is a callable of one scalar input returning two scalar outputs (bounds of its track interval), or None (when input is out of domain).

  • Its domain must be an interval (not necessarily bounded) containing 0

  • The mapping of a scalar to the lower bound of its track interval should be non decreasing and right-continuous.

The specification track can be the track map itself, returned as such after minimal checks. As a helper, track can also be

  • a positive scalar, in which case the track map is based on intervals of constant length equal to that scalar, starting at 0 and never ending

  • an increasing sequence of positive scalars, in which case the track map is based on intervals which are the consecutive pairs in that sequence (prefixed with 0)

Parameters:

  • track (float | Sequence[float] | Callable[[float], Tuple[float, float]]) – the track map specification

  • stype (type) – the type of scalars passed to the track map

class PYTOOLS.animation.animation_player_base(display, frame_per_stu=None, track=None, **ka)[source]

Bases: object

An instance of this class is a controllable matplotlib animation, created by this constructor and attached to a Figure instance stored as attribute board. The board creation is not performed in this class, so it must be performed in the constructor of a subclass, prior to invoking this constructor.

The speed of the animation is controlled by two parameters whose product determines the number of real milli-seconds per simulation time unit (stu):

  • parameter frame_per_stu: the number of frames per stu

  • parameter interval: the number of real milli-seconds per frame

Note that the interval is bounded below by the real time needed to construct and display the frames. Furthermore, below 40ms per frame, the human eye tends to loose vision persistency.

Parameters:

  • display (Callable[[float], None]) – the function which takes a simulation time and displays the corresponding (closest) frame

  • track (Callable[[float], Tuple[float, float]]) – track map decomposing the simulation domain into tracks (typically obtained by track_function())

  • frame_per_stu (float) – frame rate, in frames per simulation time (if None use animation real time rate)

  • ka – passed to the FuncAnimation constructor

show_running(b)[source]

Shows the current running state as b. This implementation raises an error.

setval(v)[source]

Sets the current frame to be displayed by the animation so that it represents simulation time v. Return a non None value if v is out of the frame range. This implementation raises an error.

class PYTOOLS.animation.widget_animation_player(displayer, fig_kw={}, **ka)[source]

Bases: app, animation_player_base

A instance of this class is a player for matplotlib animations controlled from mod:ipywidgets widgets. The animation board is created by invoking figure() with arguments fig_kw. The frame display function is obtained by calling function displayer on the board.

Parameters:

  • displayer (Callable[[Figure], Callable[[float], None]]) – the frame display factory

  • fig_kw – configuration of the animation figure

  • ka – passed to the superclass accepting the corresponding keys
